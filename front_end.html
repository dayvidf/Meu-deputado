Navegação simples

Feedback visual imediato

Decisões de Arquitetura Visual

A estrutura de front-end do Meu Deputado foi pensada com base em três pilares: componentização, escalabilidade e integração facilitada com API. Cada seção da interface corresponde a um componente independente, permitindo evolução incremental sem reestruturações profundas.

A tela de busca prioriza a escaneabilidade. O usuário consegue identificar partido, estado e nome do deputado em menos de um segundo por item listado. A paleta escura no cabeçalho cria contraste institucional, sinalizando seriedade e credibilidade ao cidadão.

Na tela de perfil, o dashboard de métricas resumidas foi posicionado imediatamente abaixo do cabeçalho institucional, permitindo uma visão geral antes da exploração detalhada. A estrutura em abas reduz a sobrecarga de informações sem esconder dados relevantes.

O resultado é uma interface clara, moderna e preparada para evoluções futuras, tanto em termos de novas funcionalidades quanto de integração com fontes de dados adicionais.

TELA 01 Busca e Listagem de Deputados Tela inicial

Função e objetivo

Ponto de entrada do app. O usuário seleciona o deputado que votou para acompanhar o trabalho dele. A tela foi projetada para permitir localização rápida por nome, partido ou estado, minimizando fricção e tempo até o conteúdo de interesse.

Componentes da tela

Elementos visuais
  • + Cabeçalho com logo e tagline institucional
  • + Campo de busca com placeholder orientador
  • + Contador de resultados encontrados
  • + Lista com avatar de iniciais, nome, partido e estado
  • + Seta de navegação para perfil do deputado
Decisões de UX
  • + Placeholders para utilização das fotos dos deputados após link com API
  • + Badges coloridos de partido facilitam escaneabilidade
  • + Estado exibido junto ao partido reduz ambiguidade
  • + Fundo escuro no header reforça credibilidade

Considerações finais
Arquitetura e Evolução

Essa estrutura de front-end foi pensada com base em componentização, escalabilidade e integração facilitada com API. Cada tela corresponde a um módulo independente, alimentado por endpoints distintos da API de Dados Abertos da Câmara dos Deputados.

A separação por abas não é apenas uma escolha visual. Ela reflete a separação lógica dos dados no back-end, facilitando o consumo incremental e a manutenção futura. Novos módulos podem ser adicionados como abas sem afetar a estrutura existente.

O resultado é uma interface clara, moderna e preparada para evoluções futuras, seja pela adição de novas métricas, integração com outras bases de dados públicas, ou expansão para plataformas adicionais.
